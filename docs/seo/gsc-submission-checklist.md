## 1. Submit the sitemap (do this once after each publish)

1. Open [Search Console](https://search.google.com/search-console) → select **`greencabinetsny.com`**.
2. Left nav → **Sitemaps**.
3. Under **Add a new sitemap**, enter: `sitemap.xml` → **Submit**.
4. Verify status flips to **Success** (refresh after ~30s). Discovered URL count should match `npm run build` sitemap output (auto-generated by `scripts/generate-sitemap.ts`).
5. If a previous entry exists and is stale, click it → ⋮ → **Remove sitemap**, then re-add.

> Optional fan-out to Bing/Yandex/Seznam/Naver: `npm run indexnow` (uses `scripts/indexnow-submit.mjs`).

## 3. Bulk-check status programmatically

Faster than the GSC UI for re-checks:

1. Visit `/admin/gsc-indexing` (admin-only).
2. Grab a token: [OAuth Playground](https://developers.google.com/oauthplayground/?scope=https://www.googleapis.com/auth/webmasters.readonly) → Authorize → Exchange → copy `access_token`.
3. Paste token; site = `sc-domain:greencabinetsny.com`; URLs are pre-filled. Click **Check indexing status**.
4. Read the **Verdict** column: PASS / PARTIAL / FAIL / NEUTRAL.

## 4. Pre-flight checks (before submitting)

Run locally to catch issues before Google does:

```bash
npm run verify:pillars     # status 200 + canonical + internal links resolve
npm run validate:jsonld    # LocalBusiness + Article schema
```

Quick manual checks per page:
- [ ] HTTP 200 (`curl -I <url>`)
- [ ] `<link rel="canonical">` matches the page URL
- [ ] No `<meta name="robots" content="noindex">`
- [ ] Not blocked by `public/robots.txt`
- [ ] Listed in `public/sitemap.xml`
- [ ] At least 2 inbound internal links from sibling pages

## 5. Follow-up cadence

- **Day 1:** Submit sitemap + request indexing for all 7.
- **Day 3:** Re-run `/admin/gsc-indexing` — most should be **PASS**.
- **Day 7:** Anything still **FAIL** or **NEUTRAL** → check **Page indexing** report → fix root cause (e.g. soft 404, redirect, duplicate canonical) → re-request.
- **Day 14:** Run `site:greencabinetsny.com/<slug>` queries to confirm SERP visibility.

---

## 6. Common failures & fixes

| Verdict / Reason | Fix |
|---|---|
| **Discovered – currently not indexed** | Add inbound internal links, improve content depth, re-request. |
| **Crawled – currently not indexed** | Quality signal — improve uniqueness vs. competing pages. Wait, don't spam re-request. |
| **Duplicate, Google chose different canonical** | Make `userCanonical` match content; check for near-duplicate body copy on sibling pages. |
| **Excluded by 'noindex' tag** | Page or parent component is rendering `<meta name="robots" content="noindex">`. Search the codebase. |
| **Blocked by robots.txt** | Edit `public/robots.txt`. |
| **Soft 404** | Page returns 200 but content looks empty/error-like. Check SSR/hydration. |
| **Server error (5xx)** | Hit URL with `curl`; check edge function / hosting logs. |
